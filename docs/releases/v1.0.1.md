A hardening and housekeeping release. No gameplay changes; saves from v1.0.0 remain compatible.

## Security

- **Save files are now treated as untrusted input.** Loading runs every field through a sanitizer that clamps all array indexes and enums to valid ranges, forces every flag to 0/1, and terminates every string, so a corrupt or hand-crafted `.sav` can produce at worst a strange game, never an out-of-bounds access. Verified by loading a save full of random bytes and playing on without a crash.
- **Fixed a latent buffer flaw in the status pane.** `wnsprintfW` returns a negative value on truncation, which could have walked the write cursor backwards past the buffer start. A guarded advance now keeps the cursor in bounds regardless of format expansion.
- **Audited every string and memory call.** No unsafe CRT functions and no unbounded Win32 string functions (`lstrcpy`, `wsprintf`, and friends) exist anywhere in the game sources; all copies and formats are bounded.

## Code structure

- Conventional header split: `gangland.h` now holds only the shared types and constants, and each module exposes its API through a matching documented header (`util.h`, `main.h`, `world.h`, `menu.h`, `actions.h`, `combat.h`, `sound.h`).
- The entry point lives alone in `entry.c`; sources include exactly the interfaces they use.
- Visual Studio project updated with all headers under the Header Files filter.
